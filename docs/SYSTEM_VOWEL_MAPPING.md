# 現在のシステムを10の日本語母音に割り当て

**作成日**: 2025年12月10日  
**目的**: 現在構築されているシステムを10の日本語母音（『あ、い、う、え、お』と『わ、ゐ、う、ゑ、を』）に割り当てる

---

## 🎯 10の母音の定義

### 見る側の体系（主観）

- **『あ』（主観）**: 個々の主観によって異なる。『う』というサービスを『あ』という主観で見る。
  - **ニーズ（需要）**: 主観として、サービスを受けたい！と考えるカスタマー
  - **シーズ（供給）**: 主観として、サービスを提供したい！と考えるデベロッパー
  - どちらも『あ』であり、それを受け取るのが『わ』
- **『い』（実行する側、最終決定）**: ボタンを押した時に実行されるプログラム
- **『う』（存在そのもの、共通事項）**: システムの存在そのもの。パブリック関連
- **『え』（選ぶ側、実践知識・選択）**: 営業マンのツール。案件を持ってくる側
- **『お』（蓄える側、経験知識）**: 社内管理システム。データを活用して分析・レポートを行う

### 見られる側の体系（客観）

- **『わ』（客観）**: それを受け取る側（客観）
  - 『あ』（主観）が持つニーズやシーズを受け取る側
  - カスタマーのニーズを受け取る側
  - 開発者のシーズを受け取る側
  - 開発者や管理者、提供者、自社
- **『ゐ』（実行される側、実行ボタン）**: 実行ボタン、実行された結果
- **『う』（存在そのもの、共通事項）**: システムの存在そのもの（両方の体系で共通）
- **『ゑ』（選ばれる側、選択された知識）**: 清掃員のユーティリティツール。案件を見て対応する側
- **『を』（蓄えられる側、蓄積知識）**: 単に蓄えられた知識、データ

## 📊 システムの流れと各母音の関係

### システムの流れ: う→あ→お→え→い

1. **『う』（存在）**: システムの存在そのもの
   - パブリックサイト、サービス一覧

2. **『あ』（主観：ニーズやシーズを持つ側）**: 
   - カスタマー：サービスを受けたい（ニーズ）
   - 開発者：サービスを提供したい（シーズ）
   - ユーザー管理、マイページ

3. **『お』（経験知識）**: データを活用して分析・レポートを行う
   - 社内管理システム、分析・レポート

4. **『え』（選択）**: どのような選択をするか？見るものの選択
   - 営業マンのツール、見積もり作成

5. **『い』（最終決定）**: 実際に走らせるプログラム
   - 購入ボタン、確定ボタン、実行プログラム

### 対比関係（うわをゑゐ の順）

- **存在**: 『う』（存在そのもの、共通事項）
- **主観と客観（ニーズとシーズ）**: 
  - 『あ』（主観：ニーズやシーズを持つ側）
    - カスタマー：サービスを受けたい（ニーズ）
    - 開発者：サービスを提供したい（シーズ）
  - 『わ』（それを受け取る側：客観）
- **蓄える側と蓄えられる側**: 『お』（蓄える側） ↔ 『を』（蓄えられる側）
- **選ぶ側と選ばれる側**: 『え』（選ぶ側） ↔ 『ゑ』（選ばれる側）
- **実行する側と実行される側**: 『い』（実行する側） ↔ 『ゐ』（実行される側）

## 🎯 まとめ

### システムの流れ（う→あ→お→え→い）

4. **『え』**: 営業マンのツール（選ぶ側、実践知識・選択）
5. **『い』**: 実行する側（最終決定、ボタンを押した時に実行されるプログラム）

この割り当てにより、システム全体の構造と各要素の役割が明確になります。
