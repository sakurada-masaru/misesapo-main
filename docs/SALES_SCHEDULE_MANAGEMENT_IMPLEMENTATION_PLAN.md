# 営業ダッシュボード スケジュール管理実装計画

## 📋 要件

営業ダッシュボードに、管理スケジュール管理と同じ機能を**スマホレスポンシブ対応**で実装する。

## 🎯 実装する機能

### 管理画面と同じ機能
1. **スケジュール一覧表示**
   - リスト表示（ID、日時、法人名、ブランド名、店舗、営業担当、清掃員、ステータス、清掃内容、操作）
   - カレンダー表示
   - ページネーション

2. **フィルター機能**
   - 店舗フィルター
   - 清掃員フィルター
   - ステータスフィルター
   - リセットボタン

3. **新規作成/編集ダイアログ**
   - 店舗・ブランド・法人名検索（ドロップダウン検索）
   - 日付・時間入力
   - 作業時間（分）
   - 営業担当者選択
   - 担当清掃員選択
   - 清掃内容選択（複数選択可能、ドロップダウン検索）
   - ステータス選択
   - 備考入力

4. **削除機能**
   - 削除確認ダイアログ

5. **新規案件アラート**
   - 未確定（draft）ステータスのスケジュール数表示

### 営業ダッシュボード特有の要件
- **スマホレスポンシブ対応**: モバイルファーストデザイン
- **営業担当者向け**: 自分が作成したスケジュールの管理
- **既存のUI/UX**: 営業ダッシュボードのデザインに合わせる

## 📁 実装ファイル

### 新規作成
- `src/pages/sales/schedules/index.html` - スケジュール管理画面（新規）
- `src/assets/css/sales-schedules.css` - スケジュール管理用CSS（スマホレスポンシブ対応）
- `src/assets/js/sales-schedules.js` - スケジュール管理用JavaScript

### 既存ファイルの修正
- `src/pages/sales/dashboard.html` - スケジュール画面へのリンクを更新

## 🎨 デザイン方針

### スマホレスポンシブ対応
- **モバイルファースト**: スマホ表示を優先
- **ブレークポイント**: 
  - モバイル: < 768px
  - タブレット: 768px - 1024px
  - PC: > 1024px

### UI/UX
- 営業ダッシュボードの既存デザインに合わせる
- 管理画面の機能は維持しつつ、営業向けに最適化
- タッチ操作に最適化（ボタンサイズ、間隔など）

## 🔄 実装ステップ

### フェーズ1: HTML構造の作成
1. `src/pages/sales/schedules/index.html` を作成
2. 管理画面のHTMLをベースに、営業ダッシュボードのデザインに合わせて調整
3. スマホレスポンシブ対応のマークアップ

### フェーズ2: CSSの実装
1. `src/assets/css/sales-schedules.css` を作成
2. 管理画面のCSSをベースに、スマホレスポンシブ対応を追加
3. 営業ダッシュボードのデザインに合わせて調整

### フェーズ3: JavaScriptの実装
1. `src/assets/js/sales-schedules.js` を作成
2. 管理画面のJavaScriptをベースに、営業向けに調整
3. スマホ操作に最適化

### フェーズ4: 統合・テスト
1. 営業ダッシュボードからのリンクを更新
2. 動作確認
3. スマホ・タブレット・PCでの表示確認

## ⚠️ 注意事項

1. **既存機能の維持**: 営業ダッシュボードの既存機能を壊さない
2. **データフィルタリング**: 営業担当者が作成したスケジュールのみ表示（必要に応じて）
3. **権限管理**: 営業担当者が編集・削除できるスケジュールの範囲を確認
4. **パフォーマンス**: スマホでのパフォーマンスを考慮

## 📝 実装詳細

### HTML構造
- 管理画面のHTMLをベースに、営業ダッシュボードのレイアウトに合わせて調整
- スマホ表示では、テーブルをカード形式に変更

### CSS
- 管理画面のCSSをベースに、レスポンシブ対応を追加
- モバイル: カード形式、横スクロールなし
- タブレット: テーブル形式、横スクロール可能
- PC: テーブル形式、全列表示

### JavaScript
- 管理画面のJavaScriptをベースに、営業向けに調整
- データフィルタリング（必要に応じて）
- スマホ操作の最適化
