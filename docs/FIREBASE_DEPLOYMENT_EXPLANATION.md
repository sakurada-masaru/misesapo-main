# Firebase Hostingデプロイの説明

## 現在の状態

### GitHub Pages
- URL: `https://sakurada-masaru.github.io/misesapo/`
- **問題点**: 
  - 管理画面の編集機能が動作しない（APIが使えない）
  - 画像アップロードができない
  - データの保存ができない

### ローカル開発サーバー
- URL: `http://localhost:5173`
- **問題点**:
  - あなたのPCでしかアクセスできない
  - 事務員が別の場所からアクセスできない
  - PCを起動していないと使えない

## Firebase Hostingでデプロイすると

### 恒久的なURL
- URL: `https://misesapo-system.web.app`
- **メリット**:
  - どこからでもアクセス可能（インターネットに接続されていれば）
  - 24時間365日利用可能
  - 事務員が自宅や外出先からでもアクセスできる

### 完全な機能が使える
- ✅ 管理画面で編集・保存ができる
- ✅ 画像をアップロードできる
- ✅ 下書き保存・確定保存ができる
- ✅ リアルタイムでデータが共有される

### データの保存場所
- **Firestore**: データベース（清掃マニュアルのテキストデータ）
- **Firebase Storage**: 画像ファイル
- **Firebase Hosting**: ウェブサイト（HTML/CSS/JS）

## ワークフロー

### 事務員・清掃管理人の方
1. `https://misesapo-system.web.app/cleaning-manual-admin.html` にアクセス
2. ログイン（管理者・清掃員・コンシェルジュ・開発者のいずれか）
3. 編集・画像アップロード
4. 「下書き保存」または「確定保存」

### あなた（技術者）
1. 管理画面にアクセス
2. 下書きを確認
3. 「下書きを確定」ボタンで本番環境に反映
4. または、下書きを削除

## デプロイの流れ

```
1. Firebase CLIをインストール
   ↓
2. Firebaseにログイン
   ↓
3. firebase deploy を実行
   ↓
4. 数分でデプロイ完了
   ↓
5. https://misesapo-system.web.app でアクセス可能に
```

## まとめ

**Firebase Hostingでデプロイ = インターネット上に公開**

- 事務員がどこからでもアクセスできる
- 編集・保存・画像アップロードがすべて動作する
- あなたが後で確認して確定できる

**現在 = まだローカル（あなたのPC）でしか使えない状態**

**デプロイ後 = インターネット上で誰でも（ログインすれば）使える状態**
