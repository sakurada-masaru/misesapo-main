## 1. パブリックサイト（未認証ユーザー向け）

### 1.1 トップページ（発注） `/index.html`

**目的**: サービス一覧を表示し、顧客が清掃サービスを選択・発注できるページ

**アクセス権限**: 公開（未認証ユーザー可）

**主要機能**:
- スプラッシュスクリーン（ロゴアニメーション）
- ヒーロー画像（カルーセル）
- サービス一覧の表示（カテゴリ別フィルター）
- サービス詳細モーダル
- カートへの追加
- カート表示（バッジ付き）

**UI要素**:
- スプラッシュスクリーン（フェードアウトアニメーション）
- ヒーロー画像（カルーセルボタン）
- カテゴリタブ（全て、厨房設備、空調設備、フロア、その他）
- サービスカード（画像、タイトル、価格）
- カートボタン（バッジ付き）
- スワイプヒント（SP版、スライドダッシュボード用）

**データ構造**:
- `src/data/service_items.json` - サービス一覧データ
- 各サービス: `id`, `title`, `category`, `price`, `image`, `forms`, `details`

**操作フロー**:
1. ページ読み込み → スプラッシュスクリーン表示（3秒）
2. サービスカードクリック → サービス詳細モーダル表示
3. 詳細入力 → カートに追加
4. カートボタンクリック → `/cart.html` へ遷移

**レスポンシブ**: SPメイン、PC対応

**関連ページ**:
- `/cart.html` - カート
- `/signin.html` - ログイン
- `/signup.html` - 新規登録

## 2. ユーザー（顧客）向けサイト

- 資料提出セクション
- 店舗情報セクション
- 定期発注設定
- カレンダー表示（PC版）
- 清掃予定一覧（SP版）

**UI要素**:
- ヘッダー（会社名、オーナー名、クイックアクションボタン）
- メッセージセクション（お知らせ、日付、タグ）
- 資料提出カード（提出ボタン、説明）
- 店舗情報カード
- 定期発注ラジオボタン（毎月、2ヶ月、3ヶ月、6ヶ月、毎年）
- カレンダー（PC版、サイドバー）
- 清掃予定リスト（SP版）
- 下矢印アニメーション（SP版、メッセージセクション下）

**データ構造**:
- 顧客情報（ハードコード）
- メッセージデータ（ハードコード）
- 清掃予定データ（ハードコード）

**操作フロー**:
1. ページ読み込み → 顧客情報・予定を表示
2. クイックアクションボタンクリック → 各ページへ遷移
3. 資料提出ボタンクリック → 資料提出ページへ遷移
4. 定期発注設定変更 → 設定保存（モック）

**レスポンシブ**: SPメイン、PC対応（レイアウト変更あり）

**関連ページ**:

## データ構造

### 主要データファイル

1. **顧客データ**: **API（DynamoDB）**
   - `GET /clients` `GET /brands` `GET /stores`
   - 構造: `{ clients: [...], total: number }`
   - 各顧客: `id`, `company_name`, `store_name`, `contact_person`, `email`, `sales_rep`, `status`, `next_date`, `cleaning_frequency`, `url`, `notes`

2. **見積もりデータ**: `src/data/estimates.json`
   - 構造: `{ estimates: [...] }`
   - 各見積もり: `id`, `client_id`, `client_name`, `store_name`, `status`, `created_date`, `sent_date`, `expiry_date`, `subtotal`, `tax`, `total`, `services`, `notes`

3. **スケジュールデータ**: `src/data/sales_schedule.json`
   - 構造: `{ schedules: [...] }`
   - 各スケジュール: `id`, `date`, `time`, `client_id`, `client_name`, `store_name`, `purpose`, `address`, `phone`, `email`, `status`, `notes`

4. **問い合わせデータ**: `src/data/new_inquiries.json`
   - 構造: `{ inquiries: [...] }`
   - 各問い合わせ: `id`, `date`, `time`, `company_name`, `contact_person`, `phone`, `email`, `message`, `status`, `priority`

5. **サービスデータ**: `src/data/service_items.json`
   - 構造: `[...]`
   - 各サービス: `id`, `title`, `category`, `price`, `image`, `forms`, `details`

6. **研修動画データ**: `src/data/training_videos.json`
   - 構造: `{ videos: [...] }`
   - 各動画: `id`, `title`, `description`, `category`, `url`, `thumbnail`, `views`, `required`

## 技術仕様

### ビルドシステム
- テンプレートエンジン: カスタム（`@layout`, `@include`, `@json`, `@jsonvar`, `@foreach`）
- ビルドスクリプト: `scripts/build.py`
- 出力先: `public/`

### データ管理
- 静的JSONファイルからデータを読み込み
- 動的ページ生成（`[id].html`）
- CSVからJSONへの自動変換（`clients.csv` → `clients.json`）

### フロントエンド
- バニラJavaScript（フレームワーク不使用）
- 動的レンダリング（JavaScriptでDOM操作）
- `localStorage` でUI状態を保存
