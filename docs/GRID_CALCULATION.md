# マイページグリッド計算

## 前提条件

### コンテナサイズ（固定・絶対に小さくしない）
- コンテナパディング: 8px
- コンテナセクションパディング: 8px
- コンテナセクション幅: calc(100% - 16px)
- フォントサイズ: clamp(0.75rem, 0.9vw, 0.95rem) = 12px〜15.2px
- **最小コンテナ幅（1マス）: 約120px以上を維持**（フォントサイズ + パディング + 余白を考慮）

### レイアウト構造
- サイドバー幅: 240px
- マイページパディング: 24px（左右）
- 利用可能幅 = 画面幅 - 240px - 48px

## 計算結果（コンテナサイズを維持）

### 1. 超大画面デスクトップ（1400px以上）
- 利用可能幅: 1400 - 240 - 48 = 1112px
- 1マスあたりの幅: 1112 / 10 = 111.2px
- **推奨: 10x6 = 60マス** ✅

### 2. 大画面デスクトップ（1280px〜1399px）
- 利用可能幅: 1280 - 240 - 48 = 992px
- 1マスあたりの幅: 992 / 9 = 110.2px
- **推奨: 9x6 = 54マス** ✅

### 3. 中画面デスクトップ（1024px〜1279px）
- 利用可能幅: 1024 - 240 - 48 = 736px
- 1マスあたりの幅: 736 / 6 = 122.7px（6列で120px以上を維持）
- **推奨: 6x6 = 36マス** ✅

### 4. 大タブレット横向き（900px〜1023px）
- 利用可能幅: 900 - 240 - 48 = 612px
- 1マスあたりの幅: 612 / 5 = 122.4px（5列で120px以上を維持）
- **推奨: 5x5 = 25マス** ✅

### 5. タブレット横向き（768px〜899px）
- 利用可能幅: 768 - 240 - 48 = 480px
- 1マスあたりの幅: 480 / 4 = 120px（4列で最小サイズを維持）
- **推奨: 4x5 = 20マス** ✅

### 6. タブレット縦向き・大きいスマホ（600px〜767px）
- 利用可能幅: 600 - 0 - 24 = 576px（サイドバー非表示想定）
- 1マスあたりの幅: 576 / 4 = 144px（4列で余裕あり）
- **推奨: 4x4 = 16マス** ✅

### 7. スマホ（〜599px）
- 利用可能幅: 375 - 0 - 24 = 351px（サイドバー非表示）
- **推奨: 1列（縦積み）** ✅

## 最適化されたブレイクポイント（コンテナサイズ維持版）

```
1. 1400px以上: 10x6 = 60マス（1マス=111.2px）
2. 1280px〜1399px: 9x6 = 54マス（1マス=110.2px）
3. 1024px〜1279px: 6x6 = 36マス（1マス=122.7px）← 変更
4. 900px〜1023px: 5x5 = 25マス（1マス=122.4px）← 変更
5. 768px〜899px: 4x5 = 20マス（1マス=120px）← 変更
6. 600px〜767px: 4x4 = 16マス（1マス=144px、サイドバー非表示）← 変更
7. 〜599px: 1列（縦積み）
```

## 重要な原則

- **コンテナサイズは絶対に小さくしない（最小120px以上を維持）**
- 画面が小さくなる場合は、グリッド数を積極的に減らす
- フォントサイズ・パディングは固定のまま
- 小さい画面ではサイドバーを非表示にすることを推奨
